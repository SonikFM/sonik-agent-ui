What changed

Visible controls

The terminal embed retains Source, Pick, Capture, More, and Right/Bottom/Fullscreen positioning controls instead of hiding the entire toolbar.

Layout persistence

The selected dock position survives reloads and iframe remounts. The terminal initially focuses fullscreen only when no saved layout exists.

Visual context recovery

Reconnect reads the saved source, route revision, context revision, status, and invalidation reason from the workspace API.

Booking navigation

Dev-open intent is stored per browser tab. After a Booking route unmount/remount, the Workbench reopens and reconnects to the same named sandbox and tmux session.

Host authorization

The exact Booking Pipe B origin is included in the Workbench production allowlist so signed host-context requests can complete.

Durability boundary

Sandbox files, tmux processes, snapshot, and context survive reconnects. Browser-only xterm scrollback and unsent keystrokes do not survive iframe destruction.

Live acceptance path

  1. Sign into Booking Pipe B and open Dev from the Agent UI speed dial.
  2. Confirm the terminal opens and Source, Pick, Capture, More, Right, Bottom, and Fullscreen controls are reachable.
  3. Confirm host context resolves to Booking rather than “unresolved host context.”
  4. Select Bottom, reload, and confirm Bottom remains selected.
  5. Create a marker in the terminal: echo durable > /workspace/.sonik/navigation-proof.txt.
  6. Navigate from the current Booking page to Reservations. Confirm Dev reopens without crashing.
  7. Run cat /workspace/.sonik/navigation-proof.txt and confirm the marker and tmux-backed shell remain available.
  8. Confirm page context is current or explicitly marked invalidated after the route change; capture a new screenshot and verify it becomes current.
  9. Press Escape to close Dev, reload, and confirm it remains closed.

Accepted limitation: route navigation reconnects the durable backend but rebuilds the iframe terminal renderer. Preserving browser scrollback and unsent input with zero socket interruption requires hoisting the embed into a persistent root layout.
